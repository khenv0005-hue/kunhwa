# 5) 운영·도입 계획

## 1. 적용 대상 및 조직
- **적용 대상 업무**: 비점오염 현장 모니터링, 수질 샘플링 채수 결정, 하천 유지관리 긴급 상황 전파.
- **적용 부서**: 물환경사업팀 및 유관 지차체 환경 담당 부서.

## 2. 운영 체계
- **운영자 구성**: 
  - (총괄) 시스템 관리자: API 키 관리 및 서버 유지보수.
  - (사용자) 현장 모니터링 요원: 대시보드 및 모바일 앱 활용 의사결정.
- **권한 및 보안**:
  - 현재 로컬/인트라넷 기반 배포로 외부 유출 차단.
  - 향후 OAuth2 기반 로그인을 도입하여 사용자별 접근 권한 차별화 예정.

## 3. 유지보수 및 지원
- **데이터 업데이트**: 기상 API 변경 및 레이어 소스 업데이트에 따른 프론트엔드 코드 정기 업데이트 (분기 1회).
- **장애 대응**: API 호출 한도 초과 시 비상 백업 소스(KMA 공공데이터 포털 등)로 자동 전환 로직 구축 계획.

## 4. 향후 고도화 로드맵
- **Phase 1 (2026.H1)**: PWA 기반 모바일 최적화 및 현장 푸시 알람 연동.
- **Phase 2 (2026.H2)**: 실시간 수질 센서(물환경정보시스템) API 연동을 통한 예측-측정 통합 분석.
- **Phase 3 (2027)**: 과거 수만 건의 기상-유역 반응 데이터를 학습시킨 머신러닝 기반 '유출 강도 예측' AI 모델 탑재.

---
> [!TIP]
> 단계별 고도화를 통해 단순 '예보 시스템'에서 '지능형 환경 통합 관제 플랫폼'으로 진화하는 것이 최종 목표입니다.

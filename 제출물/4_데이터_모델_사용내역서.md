# 4) 데이터/모델 사용 내역서

## 1. 사용 데이터 명세

| 데이터 종류 | 제공처 | 수집 방식 | 주요 활용 내용 |
| :--- | :--- | :--- | :--- |
| **글로벌 수치 예보** | Open-Meteo (WMO standard) | REST API (실시간) | KMA, GFS, ECMWF, JMA 모델 예보 데이터 |
| **기상 레이더 영상** | RainViewer (KMA 소스 포함) | Map Tiles API | 실시간 비구름 이동 경로 시각화 |
| **글로벌 기상 레이어** | Windy API | Iframe Embed | 바람장, 구름량, 장기 예측 지도 레이어 |
| **GIS 지형 데이터** | 사용자 입력 (SHP, GeoJSON) | Local File Parse | 유역 경계 및 지점 위치 매핑 |

## 2. 시스템 아키텍처 및 모델링
- **수집/익명화**: 모든 데이터는 기상청 및 글로벌 공개 API를 통해 수집되며, 개인정보를 포함하지 않는 공공 데이터 속성임.
- **판단 엔진 (Rule-based Engine)**:
    - AI 학습용 데이터셋 구축 전 단계로, 수리/기상학적 지식 기반의 **'다중 시간창 앙상블 로직'** 적용.
    - 4개 모델의 가중 평균 및 최대치를 분석하여 6단계 대응 가이드를 생성함.
- **자체 역학적 강수 보정 알고리즘**:
    - 서해상 강우 코어의 이동 속도(Wind Vector)와 태백산맥 등 지형적 상승(Orographic Lift) 효과를 수식화하여 적용. (app.js 상의 `calculatePredictedRainfall` 함수 구현)

## 3. 학습 및 저장 방식
- **학습 여부**: 현재 버전은 전문가 시스템(Expert System) 기반의 로직으로 구성되어 있으나, 향후 실제 출동 결과 및 실측 강수 데이터를 누적하여 '강수 정확도 보정 AI'로 파인튜닝할 수 있는 구조를 갖줌.
- **저장 위치**: 실시간 가용성을 위해 백엔드(FastAPI) 및 클라이언트(LocalStorage) 브라우저 메모리에 캐싱 처리함.

---
> [!IMPORTANT]
> 본 시스템은 외부 기상 데이터를 직접 복제/저장하지 않으며, 실시간으로 조회하여 분석 및 시각화하는 'Middle-ware' 역할을 수행하여 데이터 정합성을 유지합니다.

# 6) 기타사항

## 1. 팀원별 역할 분담
| 역할 | 주요 업무 내용 |
| :--- | :--- |
| **업무 총괄** | 프로젝트 기획, 비점오염 현장 요구사항 정의, 의사결정 로직 설계 |
| **개발 지원 (AI Assistant)** | FastAPI 백엔드 구축, Leaflet 기반 UI 개발, 기상 API 연동 및 알고리즘 구현 |
| **데이터 분석** | 글로벌 기상 모델별 특성 분석 및 데이터 정합성 검증 |

## 2. 사용 외부 도구 및 라이브러리
- **Framework**: FastAPI (Backend), Vanilla JS (Frontend)
- **Map/Chart**: Leaflet.js, Chart.js, Windy API, RainViewer API
- **Data Source**: Open-Meteo (KMA, GFS, ECMWF, JMA forecast models)
- **Design**: Remix Icon, CSS Flexbox/Grid (Dark Theme)

## 3. 현업 적용 시 예상 장애 요인 및 해결 방안

| 예상 장애 요인 | 해결 방안 |
| :--- | :--- |
| **네트워크 환경 취약** | 오프라인에서도 작동 가능한 PWA 캐싱 기술 적용 및 위성 통신 기기 대응 연구 |
| **데이터 API 비용 및 제한** | 공공데이터 포털의 기관 전용 API 전환을 통해 트래픽 확보 및 비용 절감 |
| **현장 요원 수용성** | 직관적인 6단계 컬러 코딩 및 간단한 UI 구성을 통해 디지털 전환 거부감 최소화 |

## 4. 참고자료 출처
- 환경부 비점오염 저감시설 설치 및 운영·관리 매뉴얼
- 한국기상청(KMA) 수치예보모델 활용 가이드
- Open-Meteo API Documentation (Open-source weather data global standard)

# 강우 기반 비점오염 수질 모니터링 출동 의사결정 시스템

이 프로젝트는 전국 어디든 마우스로 클릭하면 과거 관측 강수, 현재 레이더 에코, 미래 2개 모델 예측강수를 비교하여 비점오염 현장 모니터링 “출동” 권고 여부를 합리적으로 결정해 주는 지능형 대시보드입니다.

단순 날씨 제공 시스템에서 벗어나, [과거 3시간 누적량], [미래 3시간 예측량], [앙상블(모델)간 합의도], [레이더 접근 여부]에 기반한 점수(Score) 채점 기반 규칙 엔진이 탑재되어 있습니다.

## 프로젝트 구조

```
모니터링 의사결정 시스템/
│
├── main.py               # (Backend) FastAPI 기반 의사결정 엔진 서버
├── requirements.txt      # (Backend) 파이썬 라이브러리 목록
│
├── index.html            # (Frontend) 메인 대시보드 화면
├── css/
│   └── style.css         # (Frontend) 다크 모드 전문 대시보드 스타일
├── js/
│   └── app.js            # (Frontend) Leaflet 지도 제어, Chartjs 렌더링, API 호출
└── data/                 # 실시간 통신을 피하기 위한 캐시 파일 등 저장소 (향후 사용)
```

## 핵심 기능

1. **지도 기반 인터페이스**: Leaflet 지도 상에 마우스 클릭으로 대상 지점 좌표를 취득합니다.
2. **비동기 캐싱 구조**: 클라이언트가 외부 날씨 API(KMA 등)를 직접 호출하지 않습니다. `main.py` 백엔드 서버가 이를 주기적으로 Background Task로 모아 캐싱(Caching)해두고 클라이언트에게는 융합 판단 결과만 즉시 제공합니다. (= 트래픽 증폭방지, 외부 API 의존도 저하)
3. **앙상블 편차 분석**: 보수적 모델과 민감한 예측 모델 (Model A, B)을 시계열로 나타내고 둘 간의 편차(Agreement Score)를 산출해 신뢰도(Confidence) 항목으로 표시합니다.
4. **점수 기반 출동 판단 (Decision Engine)**:
   - 과거 3시간 집중 강우가 있었다면 +25점
   - 향후 3시간 집중 강우가 예상된다면 +25점
   - 두 모델이 전부 비를 예측한다면 +20점 (일치 실패시 감점)
   - 실시간 레이더에서 비구름이 접근 중이면 +30점
   - **총 70점 이상이면 "출동 권고", 50점 이상이면 "대기", 이하는 "미출동"**

## 실행 방법

### 1단계: 백엔드 서버 실행

시스템에 파이썬 3.8 이상이 설치되어 있어야 합니다. (VSCode 터미널 또는 CMD를 여세요)

1. 이 프로젝트 최상단 폴더로 이동합니다.
   ```bash
   cd "D:\(ai관련)\모니터링 의사결정 시스템"
   ```
2. 필요 라이브러리 설치
   ```bash
   pip install -r requirements.txt
   ```
3. 백엔드 서버 시작
   ```bash
   python main.py
   ```
   *정상 실행시 `http://0.0.0.0:8000` 로컬 서버가 구동됩니다.*

### 2단계: 프론트엔드 화면 열기

웹 서버를 띄울 수도 있지만, VSCode의 `Live Server` 확장 프로그램을 이용해서 `index.html` 파일을 여시거나, 파일을 브라우저에 그대로 드래그앤 드롭하여 여시면 됩니다.

1. `index.html`을 웹 브라우저 (Chrome 권장)로 실행합니다.
2. 화면 왼쪽 대한민국 지도에서 원하는 유역/지점을 클릭해 봅니다.
3. 우측에 **출동 판단 결과**와 하단에 막대·선 그래프로 혼합된 **시간별 강우 예측 차트**가 렌더링되는 것을 확인합니다.
